在git上合并分支

先将开发的更新提交到远程，然后拉取最新的develop和开发分支上的代码，在本地合并，推到develop分支上

  1. 提交到远程（feature-xxx)

    git add .
    git commit -m “”
    git push origin feature-xxx

  2. 拉取最新的develop的代码(feature->master->develop)

    git checkout master //在master上切去其他分支
    git pull origin master

    git checkout develop
    git pull origin develop

  3. 在develop分支上将开发分支和到develop上(develop)

    git merge –no-ff -m “merge feature-xxx” origin/feature-xxx

  4. 推到develop分支上(develop)

    git push origin develop

  5. 查看分支状态

    git branch -a //所有？远程的分支
    git branch -v //查看分支的备注信息，这里就可以看有没有同步上了

  6. 回到开发分支吧

    git checkout feature-xxx

在git上重新拉分支develop并合并开发分支

  1. 提交feature_上的分支(feature-xxx)

    git add .
    git commit -m “”
    git push origin feature-xxx

  2. 切换到master(feature-xxx)

    git checkout master //在master上切去其他分支

  3. 查看分支情况(master)

    git branch -a

  4. 删除develop分支(master)

    git branch -D develop

  5. 查看分支情况(master)

    git branch -a

  6. 拉取master(master)

    git pull origin master

  7. 拉取所有分支(master)

    git pull

  8. 切到开发分支(master)

    git checkout develop

  9. 合并分支并推送到远程(develop)

    git merge –no-ff -m “merge feature-xxx” origin/feature-xxx
    git push origin develop

  10. 回到开发分支吧(develop)

    git checkout feature-xxx

  11. (feature_xxx)

解决1099端口占有的方法

好吧，其实想关哪个端口都可以hhh

  1. 在cmd中查找端口情况

    D:\Users\li_yib>netstat -ano|findstr “1099”

  2. 结果中显示了进程号

    TCP 0.0.0.0:1099 0.0.0.0:0 LISTENING 9848
    TCP [::]:1099 [::]:0 LISTENING 9848

  3. 查看进程对应的程序名

    D:\Users\li_yib>tasklist|findstr “9848”

  4. 返回结果第一个

    java.exe 14696 Console 1 748,804

  5. 打开>任务管理器>详细信息，找到程序名进程号的程序，结束任务即可

idea设置热部署

被重新编译折磨了很久很久终于想起来解决这个问题了，搞好了真的神清气爽哈哈哈哈

配置步骤

  1. 在Run/Debug Configuration中选artifact为 ：war exploded
  2. 在Run/Debug Configuration中server页面
    a. On ‘Update’action: Update classes and resources
    b. On frame deactivation: Update classes and resources
  3. 设置Debugger-HotSwap
    在setting界面，打开Debugger-HotSwap选项，确保勾选了Build project before reloading classes，同时选择Reload classes after compilation为Always。这样我们在编译某个修改了的java文件之后，就会利用HotSwap机制reload class，而Build project before reloading classes就确保了其他修改过的文件一起同步到部署目录。
    图片: https://upload-images.jianshu.io/upload_images/5321750-a7224ea20ba87cf6.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/700
  4. 点击debug运行

    这样不管是改Java文件还是修改jsp，只要失去焦点就会自动部署了，就不用重新运行了ahhh

  5. 题外话
    在配置tomcat的时候把打开的链接改成这次开发的链接，就不用每次都输入了
